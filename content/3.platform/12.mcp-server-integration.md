## Overview

The Sulala Agent supports **MCP (Model Context Protocol)** servers as an alternative or complement to OAuth Integrations. MCP servers expose tools (e.g. search YouTube, send a tweet) that the agent can call. Each tool is registered with a prefixed name: `mcp_<serverName>_<toolName>` so they donÔÇÖt clash with built-in or skill tools.

- **Config:** Dashboard Ôćĺ **Integrations** Ôćĺ **MCP Servers** tab, or `~/.sulala/mcp.json`, or env `MCP_SERVERS`.
- **Discovery:** Find community servers at [mcpservers.org](https://mcpservers.org/).

## How it works

1. You add one or more MCP servers (name, command, args, optional env for API keys).
2. On startup (and after saving config from the dashboard), the agent spawns each server over stdio, lists its tools, and registers them as agent tools with the `mcp_<name>_` prefix.
3. The model sees these tools in the same tool list as built-in and plugin tools; when it calls one, the agent forwards the call to the MCP server and returns the result.

## Configuration

### Via Dashboard

1. Open **Integrations** Ôćĺ **MCP Servers**.
2. Click **Add server** and set:
   - **Name** ÔÇö Short identifier (e.g. `youtube`, `twitter`). Used in tool names: `mcp_youtube_...`.
   - **Command** ÔÇö Executable (e.g. `npx`, `node`).
   - **Args** ÔÇö Array of arguments (e.g. `-y`, `zubeid-youtube-mcp-server`), comma-separated in the form.
   - **Env** ÔÇö Optional key/value pairs (e.g. `YOUTUBE_API_KEY`). Values are stored in `~/.sulala/mcp.json` and redacted in the UI as `***`.
3. Save. Config is written to `~/.sulala/mcp.json` and tools are refreshed immediately.

### Via file: `~/.sulala/mcp.json`

Create or edit `~/.sulala/mcp.json`:

```json
{
  "servers": [
    {
      "name": "youtube",
      "command": "npx",
      "args": ["-y", "zubeid-youtube-mcp-server"],
      "env": {
        "YOUTUBE_API_KEY": "your-api-key"
      }
    }
  ]
}
```

Restart the agent (or use the dashboard save) so the agent reloads config and re-registers tools.

### Via environment: `MCP_SERVERS`

For automation or non-interactive setups, set `MCP_SERVERS` to a JSON array of server objects (same shape as above):

```bash
export MCP_SERVERS='[{"name":"youtube","command":"npx","args":["-y","zubeid-youtube-mcp-server"],"env":{"YOUTUBE_API_KEY":"..."}}]'
```

If `MCP_SERVERS` is set, it takes precedence over `~/.sulala/mcp.json`.

## Agent behaviour

- **Tool naming:** Tools are exposed as `mcp_<serverName>_<toolName>`, with `serverName` normalized to lowercase and non-alphanumeric characters replaced by `_`.
- **Policy:** MCP tools are subject to the same allowlist and profile (`AGENT_TOOL_ALLOWLIST`, `AGENT_TOOL_PROFILE`) as other tools.
- **Prompt:** The agentÔÇÖs system prompt instructs the model to prefer integration tools for a given provider when a connection exists (e.g. Bluesky, X/Twitter), and to use MCP tools when no matching integration is available or when the action fits an MCP tool (e.g. `mcp_twitter_post_tweet`).

## Gateway API

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/mcp/config` | Get MCP servers config for display. Env values are redacted as `***`. |
| PUT | `/api/mcp/config` | Save MCP servers. Body: `{ "servers": [ { "name", "command", "args?", "env?" } ] }`. Env values sent as `***` are preserved from existing config. Triggers tool refresh. |

Same auth as other gateway routes (e.g. `X-Api-Key` when `GATEWAY_API_KEY` is set).

## Example: YouTube MCP

1. Get a YouTube Data API key from Google Cloud Console.
2. In Dashboard Ôćĺ Integrations Ôćĺ MCP Servers, add:
   - Name: `youtube`
   - Command: `npx`
   - Args: `-y`, `zubeid-youtube-mcp-server`
   - Env: `YOUTUBE_API_KEY` = your key
3. Save. Tools such as `mcp_youtube_search` (exact name depends on the server) will appear; the agent can then use them when answering questions about YouTube.

## Troubleshooting

- **Server fails to load:** Check agent logs for `Failed to load MCP server "<name>"`. Ensure the command is on PATH and args are correct; for npx packages, ensure Node.js and network are available.
- **No tools listed:** Some MCP servers expose resources or prompts only; the agent only registers **tools**. If the server has no tools, youÔÇÖll see `MCP server "<name>" has no tools` in the logs.
- **Config not applied:** If you edit `mcp.json` by hand, restart the agent or trigger a save from the dashboard so the agent reloads config and calls `refreshMcpTools`.
