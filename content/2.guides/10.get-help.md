## Overview

Use the following to get help, report issues, or contribute.

## Documentation

- **This site** — [Getting started](/getting-started/quick-start), [Platform](/platform/agent) (Agent, Integrations, Store), [Guides](/guides/deployment), and [Troubleshooting](/troubleshooting/common-issues) cover setup, configuration, and common problems.
- **FAQ** — [FAQ](/troubleshooting/faq) answers frequent questions (e.g. “Do I need Integrations?”, “Where are tokens stored?”, “How do I add my own skill?”).

## Bugs and feature requests

- **GitHub Issues** — If the project is hosted on GitHub, open an issue in the relevant repository (e.g. Agent, Integrations, Store or a monorepo). Include:
  - What you did (steps to reproduce).
  - What you expected.
  - What actually happened (error message, logs if safe to share).
  - Your environment (OS, Node version, how you run the agent/store/integrations).
- **Feature requests** — Same place as bugs; describe the use case and desired behaviour.

## Contributing

- **Contribution guide** — See [Contribution guide](/guides/contribution-guide) for running from source, where to change code, lint and tests, and how to submit skills, integrations, or docs.
- **Pull requests** — Welcome for docs, bug fixes, and features. Follow the repo’s guidelines and link any related issues.

## Community

If the project has a **Discord**, **GitHub Discussions**, or other community channel, the maintainers may link it in the repository README or on the project website. Check the main repo for an “Community” or “Discuss” link.

## Summary

| Need | Where |
|------|--------|
| How to do X | Docs (this site), FAQ |
| Something’s broken | Troubleshooting, then GitHub Issues |
| Feature idea | GitHub Issues or Discussions |
| Contribute code/docs | [Contribution guide](/guides/contribution-guide) |
