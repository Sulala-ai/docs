## Overview

The **SulalaHub Store** is a public skill store for the [Sulala Agent](/platform/agent). You host this app and point the agent at it so users can browse and install skills from the registry. It supports free and paid skills, creator accounts, and optional admin moderation.

## Run locally

```bash
cd store
pnpm install
cp .env.example .env   # set NEXT_PUBLIC_STORE_URL and optionally ADMIN_SECRET
pnpm dev
```

Open http://localhost:3000. The storefront lists skills; each skill page has install instructions.

## Use from the agent

1. Deploy the store (e.g. Vercel) and set its **base URL** (e.g. `https://your-store.vercel.app`).
2. In your Sulala setup, set:
   ```bash
   SKILLS_REGISTRY_URL=https://your-store.vercel.app/api/sulalahub/registry
   ```
3. Install a skill:
   - **Dashboard:** Skills → Add skill from registry → choose the skill.
   - **CLI:** `sulala skill install <slug>` (e.g. `sulala skill install news`).

The agent fetches the registry from `SKILLS_REGISTRY_URL` and each skill’s content from the `url` field in the registry (e.g. `https://your-store.vercel.app/api/sulalahub/skills/news`).

## Registry contract (agent compatibility)

- **GET /api/sulalahub/registry** → `{ "skills": [ { "slug", "name", "description", "version?", "url" } ] }`.  
  `url` = full URL to fetch the skill content (Markdown).
- **GET /api/sulalahub/skills/:slug** → raw Markdown body of the skill.  
  Paid skills may require `?license=TOKEN`.

No change needed in the agent; it already uses this contract.

## Adding skills (store data)

- **Registry:** Edit `data/registry.json`. Add entries with `slug`, `name`, `description`, and optional `version`.
- **Skill content:** Store each skill as a folder: `data/skills/<slug>/README.md` (and optionally `tools.yaml`). The API serves the main content from `README.md`.
- **Store-only metadata:** Edit `data/store-meta.json` for `category`, `tags`, `featured`, `addedAt` per slug (used for catalog search, filters, Featured and Recently added).

**Sync from repo:** From repo root: `pnpm --filter store sync-registry`, or from `store/`: `pnpm sync-registry`. Copies the main agent registry into `store/data/`. A GitHub Action can run this on push so the deployed hub stays in sync.

## Submitting a skill

- **Form:** Go to `/submit`, fill slug, name, description, and skill Markdown. Submissions are stored in `data/submissions.json` (moderation queue). On Vercel the filesystem is read-only, so the form may direct users to submit via GitHub.
- **Submit via GitHub:** Set `NEXT_PUBLIC_SUBMIT_GITHUB_ISSUE_URL` to a GitHub issue template URL so users can submit via an issue.
- **Publish from agent:** Creators can generate a publish API key in the store (My skills → Create publish API key) and add it in the agent dashboard (Settings → Portal → Store publish API key). Submissions sent with that key are linked to their account.

## Creator accounts (optional)

- **Sign in with Google** via [Clerk](https://clerk.com). No account needed to browse or submit.
- **My skills:** When signed in, use **My skills** (`/my-skills`) to see skills you’ve submitted and their status (e.g. Pending review / Published).
- Configure Clerk env vars in `.env` (see `.env.example`).

## Admin

- **URL:** `/admin`. Set **`ADMIN_SECRET`** in the environment to protect it.
- **Login:** Open `/admin`, enter the secret. You can then approve/reject submissions and remove skills from the store.
- On Vercel, writes to `data/` are not persistent; use admin on a self-hosted instance where the app can write to disk, or use a database/blob store for submissions and registry.

## Paid skills (optional)

Skills can be **free** or **paid**. Creators connect a Stripe account (Stripe Connect) to receive payouts; buyers purchase on the store and get a license token to install the skill.

- **Setup:** Add `STRIPE_SECRET_KEY` and `STRIPE_WEBHOOK_SECRET`; run the paid-skills migration (see store `README` and `supabase/migrations/`).
- **Flow:** Creators choose Free or Paid on submit; after approval they connect Stripe and set price in **My skills**. Buyers click **Buy**, complete Stripe Checkout, then get an install URL that includes their license.
- **Registry:** Paid skills still appear in the public registry; content is gated by `?license=TOKEN` when fetching skill Markdown.

## Deploy

- **Vercel:** Connect the repo and deploy. Set **`NEXT_PUBLIC_STORE_URL`** to your production URL so install instructions and registry `url` fields use the correct base.
- **Elsewhere:** `pnpm build`, then `pnpm start`. Set `NEXT_PUBLIC_STORE_URL` to your public base URL.

For API contract and roadmap phases, see the store’s **ROADMAP.md**.
