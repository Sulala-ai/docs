## Overview

Copy `integrations/.env.example` to `integrations/.env` and set the variables you need. Keep secrets out of git.

## Server

| Variable | Default | Description |
|----------|---------|-------------|
| `INTEGRATIONS_PORT` | 1717 | Port the integrations server listens on. |
| `INTEGRATIONS_HOST` | 127.0.0.1 | Bind address (if not in example, often defaults to 127.0.0.1). |

## Callback and store

| Variable | Description |
|----------|-------------|
| **INTEGRATIONS_CALLBACK_BASE_URL** | Base URL where this service is reachable (no trailing slash). Used to build the OAuth callback URL. Full callback = `INTEGRATIONS_CALLBACK_BASE_URL/oauth/callback`. **Register that exact URL** in each OAuth app (GitHub, Google, Asana, etc.). Must match the host/port the user’s browser can reach (e.g. `http://localhost:1717` when running locally). |
| **INTEGRATIONS_STORE** | `file` (default) or `database`. Use `database` when running for cloud/portal (multi-tenant). |
| **INTEGRATIONS_DATA_PATH** | When store is `file`: path to JSON file (default `~/.sulala/integrations.json`). |
| **SUPABASE_URL**, **SUPABASE_SERVICE_ROLE_KEY** | When store is `database`: Supabase project URL and service role key. Run migrations (e.g. `20250303100000_connection_credentials.sql`, `20250303100001_bluesky_sessions.sql`). |
| **INTEGRATIONS_API_SECRET** | When running in cloud (not localhost): set a secret so only the portal (or trusted apps) can call this API. Portal must set the same value and send it as `Authorization: Bearer <secret>` or `X-Integrations-Api-Secret`. |

## Provider credentials (OAuth apps)

For each provider you want to support, create an OAuth app in that provider’s developer console and set the callback URL to **`INTEGRATIONS_CALLBACK_BASE_URL/oauth/callback`**.

| Provider | Env vars | Where to create OAuth app |
|----------|----------|---------------------------|
| **GitHub** | `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` | https://github.com/settings/developers → OAuth Apps → New OAuth App |
| **Google** (Gmail, Calendar, Drive, Contacts, Docs, Sheets, Slides) | `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` | https://console.cloud.google.com/apis/credentials → OAuth 2.0 Client ID (Web). Add redirect URI; enable Gmail API, Calendar API, Drive API, People API, Docs API, Sheets API, Slides API as needed. |
| **Airtable** | `AIRTABLE_CLIENT_ID`, `AIRTABLE_CLIENT_SECRET` | https://airtable.com/create/oauth. Redirect URL = callback above. Scopes: data.records:read, data.records:write, schema.bases:read. |
| **Asana** | `ASANA_CLIENT_ID`, `ASANA_CLIENT_SECRET` | https://app.asana.com/0/my-apps → Create new app → OAuth tab, Redirect URL = callback above. |
| **Facebook** | `FACEBOOK_CLIENT_ID`, `FACEBOOK_CLIENT_SECRET` | https://developers.facebook.com/apps → Facebook Login → Valid OAuth Redirect URIs = callback above. |
| **Notion** | `NOTION_CLIENT_ID`, `NOTION_CLIENT_SECRET` | https://www.notion.so/my-integrations → Public integration, Redirect URI = callback above. |
| **Linear** | `LINEAR_CLIENT_ID`, `LINEAR_CLIENT_SECRET` | https://linear.app/settings/api → OAuth applications, Redirect URL = callback above. Scopes: read, write. |
| **Slack** | `SLACK_CLIENT_ID`, `SLACK_CLIENT_SECRET` | https://api.slack.com/apps → Create app → OAuth & Permissions, Redirect URL = callback above. |
| **Zoom** | `ZOOM_CLIENT_ID`, `ZOOM_CLIENT_SECRET` | https://marketplace.zoom.us/develop/create → OAuth app, Redirect URL = callback above. Add scopes: meeting:read:list_meetings, meeting:read:meeting, meeting:write:meeting. |
| **Dropbox** | `DROPBOX_CLIENT_ID`, `DROPBOX_CLIENT_SECRET` | https://www.dropbox.com/developers/apps → Create app. Redirect URL = callback above. Use App secret from Settings (not the long “Generated access token”). Scopes: files.metadata.read, files.content.read, files.content.write. |
| **Jira (Atlassian)** | `JIRA_CLIENT_ID`, `JIRA_CLIENT_SECRET` | https://developer.atlassian.com/console/myapps/ → Create app, add Jira API. Redirect URL = callback above. Scopes: read:jira-work, write:jira-work. |
| **Figma** | `FIGMA_CLIENT_ID`, `FIGMA_CLIENT_SECRET` | https://www.figma.com/developers/apps → Create OAuth app. Redirect URL = callback above. Scopes: file_content:read, file_metadata:read, projects:read. |
| **Bluesky** | (optional) `BLUESKY_DEFAULT_PDS_URL` | No client_id/secret (public client). Client metadata at `INTEGRATIONS_CALLBACK_BASE_URL/oauth/bluesky-client-metadata.json`. Optional: `BLUESKY_DEFAULT_PDS_URL=https://bsky.social` for auth discovery. |

Google Slides uses the same `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`; enable Google Slides API in Cloud Console.
