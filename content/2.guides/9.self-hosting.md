## Overview

You can run the **Agent**, **Integrations**, and **Store** entirely on your own servers or VPS. This page gives a checklist and network outline; for per-app deployment details see [Deployment](/guides/deployment).

## Architecture (self-hosted)

```
[Your users / devices]
        │
        ▼
[Agent]  ←── .env (PORT, HOST, DB_PATH, SKILLS_REGISTRY_URL, PORTAL_* or INTEGRATIONS_URL)
        │
        ├──► [Integrations]  ←── .env (PORT, CALLBACK_BASE_URL, provider client IDs/secrets, INTEGRATIONS_API_SECRET)
        │         (OAuth tokens; agent calls here for connectionId → token)
        │
        └──► [Store]  ←── .env (NEXT_PUBLIC_STORE_URL, Clerk, Supabase, Stripe, ADMIN_SECRET)
                  (Registry + skill content; agent fetches registry and skills from here)
```

- **Agent** runs on one or more machines (e.g. each user’s machine, or a shared server if you expose the gateway). It needs to reach Integrations (for OAuth tokens) and Store (for registry/skills) over HTTPS in production.
- **Integrations** must be reachable by the user’s browser (for OAuth redirects) and by the Agent (for token requests). Set `INTEGRATIONS_CALLBACK_BASE_URL` to the public URL of the Integrations service.
- **Store** must be reachable by the user’s browser (catalog, submit, admin) and by the Agent (GET registry and GET skill content). Set `NEXT_PUBLIC_STORE_URL` to the public URL of the Store.

## Checklist

### Agent

- [ ] Install and run (e.g. `npm start` or daemon). See [Deployment](/guides/deployment#agent).
- [ ] Set `HOST` and `PORT` if not default. For remote access, use a reverse proxy (e.g. nginx) and keep `GATEWAY_API_KEY` set.
- [ ] Set `SKILLS_REGISTRY_URL` to your Store’s registry URL (e.g. `https://your-store.example.com/api/sulalahub/registry`).
- [ ] Set `PORTAL_GATEWAY_URL` and `PORTAL_API_KEY` (or `INTEGRATIONS_URL`) to your Integrations endpoint so the agent can get OAuth tokens.
- [ ] Configure AI (Ollama local or API keys for OpenAI/Claude/etc.). See [Agent configuration](/platform/agent-config).

### Integrations

- [ ] Deploy the Integrations app so it’s reachable at a public HTTPS URL.
- [ ] Set `INTEGRATIONS_CALLBACK_BASE_URL` to that URL (e.g. `https://integrations.example.com`). No trailing slash.
- [ ] In each OAuth provider’s app settings, set the callback URL to `INTEGRATIONS_CALLBACK_BASE_URL/oauth/callback`.
- [ ] Set `INTEGRATIONS_API_SECRET` and configure the Agent (or Portal) to send that secret when calling the Integrations API.
- [ ] For production/multi-tenant, consider `INTEGRATIONS_STORE=database` and Supabase (or your DB). See [Integrations configuration](/platform/integrations-config).

### Store

- [ ] Deploy the Store (e.g. Next.js on a Node server or Vercel). See [Deployment](/guides/deployment#store).
- [ ] Set `NEXT_PUBLIC_STORE_URL` to the public URL (e.g. `https://hub.example.com`).
- [ ] Set `ADMIN_SECRET` to protect `/admin`.
- [ ] Optional: Clerk (creator accounts), Supabase (users, paid skills), Stripe (paid skills). See [Store configuration](/platform/store-config).

## Network

- **Agent → Integrations:** HTTPS. Agent needs to call `POST /connections/:id/use` and possibly list connections. Firewall must allow outbound from Agent to Integrations.
- **Agent → Store:** HTTPS. Agent needs to GET registry and GET skill content. Firewall must allow outbound from Agent to Store.
- **Browser → Integrations:** HTTPS. OAuth redirects. Must be reachable from users’ browsers.
- **Browser → Store:** HTTPS. Catalog, submit, admin, My skills. Must be reachable from users’ browsers.

## Security

- Use **HTTPS** for all public endpoints (Integrations, Store). Prefer TLS 1.2+.
- Keep **secrets** in `.env` (or your secrets manager). Never commit them.
- Restrict **Agent** to localhost unless you intentionally expose it; then use a reverse proxy and `GATEWAY_API_KEY`.
- See [Security](/platform/security) for token storage, callback URLs, and admin protection.
