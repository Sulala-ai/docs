## Overview

The **Integrations** service is a standalone OAuth connections server. It lets users connect accounts (Gmail, GitHub, Asana, Slack, etc.) once; any app—including the Sulala Agent—can then use those connections via a `connectionId` without storing tokens themselves.

- **Default URL:** http://127.0.0.1:1717
- **Reusable:** Same service can back the agent dashboard, the portal, or other apps.

## Run locally

```bash
cd integrations
npm install
npm run dev
```

## Environment

| Variable | Default | Description |
|----------|---------|-------------|
| `INTEGRATIONS_PORT` | 1717 | Server port |
| `INTEGRATIONS_HOST` | 127.0.0.1 | Bind address |
| `INTEGRATIONS_DATA_PATH` | ~/.sulala/integrations.json | Where tokens are stored |
| `INTEGRATIONS_CALLBACK_BASE_URL` | http://localhost:1717 | Must match where the server is reachable for OAuth callbacks |
| `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET` | — | GitHub OAuth App credentials |
| (Similarly for other providers) | — | Per-provider client ID/secret env vars |

## API

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Health check |
| GET | `/providers` | List supported providers: `{ providers: [ { id, name, iconUrl, description }, ... ] }` |
| GET | `/integrations` | List all providers with their connections (id, name, iconUrl, description, connections). Use this so the UI doesn’t need to merge `/providers` and `/connections`. Requires API secret when set. |
| GET | `/connections` | List connections (tokens redacted). `?provider=github` |
| GET | `/connections/:id` | One connection (redacted) |
| POST | `/connect/:provider` | Start OAuth. Body: `{ "redirect_success": "https://..." }`. Returns `{ authUrl, state, connectionId }`. Redirect user to `authUrl`. |
| GET | `/oauth/callback` | OAuth callback (used by provider). Redirects to `redirect_success` with `?connectionId=...` or `?error=...` |
| POST | `/connections/:id/use` | **For the agent:** get access token for this connection. Returns `{ connectionId, provider, accessToken, scopes }`. |
| DELETE | `/connections/:id` | Disconnect (delete stored tokens) |

## Flow

1. App (e.g. dashboard) calls `POST /connect/github` with `redirect_success` (e.g. your app’s “connections” page).
2. Response: `{ authUrl, connectionId }`. Open `authUrl` in the browser.
3. User signs in with the provider; provider redirects to this service’s `/oauth/callback`.
4. Service exchanges code for token, stores it, redirects to `redirect_success?connectionId=...`.
5. App stores `connectionId` (e.g. in skill config or agent settings). No token is stored in the app.
6. When the agent needs to call the provider, it calls `POST /connections/:id/use` with that `connectionId` and uses the returned `accessToken`.

## Using from the agent

1. Run the integrations service (or use a shared instance).
2. In the agent dashboard, set **Portal / Integrations** URL and API key so the agent can call the integrations API.
3. In the dashboard **Integrations** (or **Settings → Portal**) area, connect providers; the UI will redirect you through OAuth and save `connectionId`s.
4. Agent tools (e.g. Gmail, GitHub, Slack) use `list_integrations_connections` and `getConnectionToken(connection_id)` to obtain tokens at runtime.

## Adding providers

Add a new file under `integrations/src/providers/` (e.g. `gmail.ts`) with auth URL and token exchange, then register it in `src/providers/index.ts`. Use env vars for client ID/secret (e.g. `GMAIL_CLIENT_ID`, `GMAIL_CLIENT_SECRET`).

## Supported integrations (agent tools)

The agent can use connected accounts for many providers. Examples:

- **GitHub** — list repos, create issue/PR, get file, comments
- **Gmail** — send, list messages, get message, archive (with search `q`)
- **Google Calendar** — list events, create/update/delete event, list calendars
- **Google Drive** — list, upload, download, create folder
- **Slack** — list channels, send message, channel history, reactions, list users
- **Notion** — search, get/create/update page, query/create database
- **Linear, Jira, Airtable, Zoom, Dropbox, Stripe, Figma, Discord**, and more

See the **integrations** repo `INTEGRATIONS_ROADMAP.md` for the full list and how to add or extend agent tools.
