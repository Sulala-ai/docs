## Config file path

The agent uses a **single JSON config file** for skill enable/disable and per-skill options (no database). Path resolution:

| Priority | Condition | Path used |
|----------|-----------|-----------|
| 1 | `SULALA_CONFIG_PATH` is set | That path (`~` expanded in code) |
| 2 | `.sulala/config.json` exists in current working directory | `./.sulala/config.json` |
| 3 | Otherwise | `~/.sulala/config.json` |

So by default: `~/.sulala/config.json`. To use a project-local config, create `.sulala/config.json` in the project and run the agent from that directory, or set `SULALA_CONFIG_PATH` to that file.

## Environment variables

Copy `agent/.env.example` to `agent/.env` and adjust. Keep secrets out of git.

### Gateway

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | 2026 | Gateway port. |
| `HOST` | 127.0.0.1 | Bind address. |
| `DB_PATH` | ./data/sulala.db | SQLite database path (relative to project root). |
| `GATEWAY_API_KEY` | — | Optional. If set, API requests must send `X-Api-Key` header or `?api_key=`. |

### Webhooks and debug

| Variable | Description |
|----------|-------------|
| `WEBHOOK_URL` | Optional; single webhook URL for `file_event` and `task_done`. |
| `WEBHOOK_URLS` | Optional; comma-separated URLs (e.g. `https://a.com/hook,https://b.com/hook`). |
| `WEBHOOK_SECRET` | Optional; secret for webhook signing. |
| `DEBUG` | Optional; set to 1 to enable debug logs. |
| `RATE_LIMIT_MAX` | Max requests per window (default 100). |
| `RATE_LIMIT_WINDOW_MS` | Window in ms (default 60000). |

### Agent run

| Variable | Default | Description |
|----------|---------|-------------|
| `AGENT_TIMEOUT_MS` | 300000 | Max run time per turn (ms). 0 = no timeout. Request body can override with `timeout_ms`. |
| `AGENT_MAX_CONTEXT_TOKENS` | 70000 | Estimated token cap; oldest messages dropped until under. |
| `AGENT_USE_PI` | — | Set to 0 to disable Pi coding-agent runtime and use default loop. |
| `AGENT_SHARED_MEMORY_KEY` | — | Optional; used for shared memory across runs. |

### Skills

| Variable | Default | Description |
|----------|---------|-------------|
| `SULALA_SKILLS_DIR` | ~/.sulala/workspace/skills | Directory for user/workspace skills; subdirs = skills (README.md or SKILL.md). `~` is expanded. |
| `SULALA_CONFIG_PATH` | ~/.sulala/config.json | Override config file path (skill enable/disable, per-skill options). |
| `AGENT_WORKSPACE_ROOT` | — | If set, `read_file` / `write_file` are scoped to this path. |
| `SULALA_WORKSPACE_SKILLS_MY_DIR` | ~/.sulala/workspace/skills/my | Directory for “my” workspace skills (e.g. agent-created skills). |
| `SKILLS_REGISTRY_URL` | — | Single URL for skills registry (list + install). e.g. `https://hub.sulala.ai` or `https://hub.sulala.ai/api/sulalahub/registry`. When serving as hub, base for skill links is derived from this. |
| `SULALA_LOG_SKILL_ENV` | — | When 1, log skill env (e.g. for debugging). |
| `AGENT_EXECUTION_PREVIEW` | — | When 1, high-risk tools (write_file, run_command, post_to_bluesky) require approval in Dashboard → Pending. |
| `SKILLS_EXTRA_DIRS` | — | Extra directories to load skills from. |
| `SKILLS_WATCH` | — | If set, watch skill dirs and emit `skills_changed` via WebSocket. |
| `STORE_PUBLISH_API_KEY` | — | Optional. If your store requires auth for submissions, set this (or use per-creator key from store). |

### Integrations (OAuth: GitHub, Gmail, Asana, Linear, etc.)

**Option A — Portal (recommended)**  
Use connections from [Portal](https://portal.sulala.ai) → API Keys. Get an API key from https://portal.sulala.ai/api-keys and set:

| Variable | Description |
|----------|-------------|
| `PORTAL_GATEWAY_URL` | URL of the Portal gateway, e.g. `https://portal.sulala.ai/api/gateway`. |
| `PORTAL_API_KEY` | API key from Portal (e.g. `sk_live_...`). |

**Option B — Direct integrations service**  
For local or single-tenant only: point the agent at your own integrations service (e.g. http://localhost:1717). When set and Portal is not set, the agent talks to integrations directly.

| Variable | Description |
|----------|-------------|
| `INTEGRATIONS_URL` | URL of the integrations service (e.g. `http://localhost:1717`). Connect providers in Dashboard → Integrations first. |

### OAuth “Connect with Sulala” (dashboard button)

Register the agent in Portal with `redirect_uri = http://localhost:3000/api/oauth/callback` (or your dashboard URL), then set:

| Variable | Description |
|----------|-------------|
| `PORTAL_OAUTH_CLIENT_ID` | OAuth client ID from Portal. |
| `PORTAL_OAUTH_CLIENT_SECRET` | OAuth client secret from Portal. |
| `DEFAULT_PORTAL_GATEWAY_URL` | Same as Portal gateway URL, e.g. `https://portal.sulala.ai/api/gateway`. |

### AI (optional)

Set via dashboard onboarding or in `.env`. Ollama works with no key.

| Variable | Description |
|----------|-------------|
| `OPENAI_API_KEY` | OpenAI API key. |
| `ANTHROPIC_API_KEY` | Claude (Anthropic) API key. |
| `GOOGLE_GEMINI_API_KEY` or `GEMINI_API_KEY` | Google Gemini API key. |
| `OPENROUTER_API_KEY` | OpenRouter API key (unified multi-model). |
| `OLLAMA_BASE_URL` | Ollama base URL (default http://localhost:11434). |
| `AI_DEFAULT_PROVIDER` | Default provider: `ollama`, `openai`, `claude`, `gemini`, `openrouter`. |

### Watcher (optional)

| Variable | Description |
|----------|-------------|
| `WATCH_FOLDERS` | Comma-separated paths to watch; or use `config/watched.json` with `folders` array; both are merged. |

### Other agent behaviour (optional)

| Variable | Description |
|----------|-------------|
| `AGENT_MAX_HISTORY_MESSAGES` | Max messages kept in context; older dropped. |
| `AGENT_TOOL_PROFILE` | `full` \| `messaging` \| `coding` \| `minimal`. Restricts which tools are available. |
| `AGENT_TOOL_ALLOWLIST` | Comma-separated tool names; only these tools are available. |
| `AGENT_TOOL_RETRY_COUNT` | Retries for failed tool calls (default 2). |

## Disabling a skill

In the config JSON, set `skills.entries.<name>.enabled: false` for that skill. The file is the one at the path chosen by `SULALA_CONFIG_PATH` or the default above.
