## Overview

- **Secrets** live in `.env` (or environment); never commit them to the repo.
- **Agent** binds to `127.0.0.1` by default; no internet exposure.
- **OAuth tokens** are stored only in the Integrations service; the agent uses a `connectionId` and fetches tokens at runtime.
- **Store** can protect admin and submission APIs with secrets; paid skill content is gated by license tokens.

## Agent

- **Binding:** Gateway and services bind to **127.0.0.1** only (configurable via `HOST`). No public exposure unless you explicitly change this.
- **Secrets:** API keys (OpenAI, Claude, Gemini, OpenRouter, etc.) and optional `GATEWAY_API_KEY` are read from `.env`. Use onboarding or Settings to set them; never put them in code or in git.
- **Config:** Skill config (enabled/disabled, per-skill options) is in a single JSON file (`~/.sulala/config.json` or project `.sulala/config.json`). No database; keep that file out of public repos if it contains sensitive options.
- **Tools:** `run_command` can be restricted via `ALLOWED_BINARIES`; use it to limit which executables the agent may run. `read_file` / `write_file` are scoped to `AGENT_WORKSPACE_ROOT` when set.

## Integrations (OAuth)

- **Token storage:** OAuth access (and refresh) tokens are stored **only** in the Integrations service (e.g. `~/.sulala/integrations.json` or the path set by `INTEGRATIONS_DATA_PATH`). The agent never stores tokens; it stores only a `connectionId` and calls `POST /connections/:id/use` to get a short-lived token when needed.
- **Callback URL:** Set `INTEGRATIONS_CALLBACK_BASE_URL` to the exact URL where the service is reachable. Register this same URL in each provider’s OAuth app to avoid redirect or token leakage to wrong apps.
- **Credentials:** Provider client IDs and secrets (e.g. `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`) belong in the Integrations service `.env`, not in the agent or frontend.

## Store

- **Admin:** Protect `/admin` with **`ADMIN_SECRET`**. Without it, admin APIs may return 401; with it, anyone who knows the secret can approve/reject submissions and remove skills. Keep it secret and use HTTPS in production.
- **Submissions:** Optional **`SUBMIT_API_KEY`** or per-creator publish keys (created in My skills) restrict who can submit. If neither is set, the store may accept unauthenticated submissions (e.g. web form).
- **Paid skills:** License tokens are issued after purchase. Skill content for paid skills is gated by `?license=TOKEN` or session; do not share license tokens.
- **Creator accounts:** Sign-in (e.g. Clerk) is for identity only; configure Clerk and webhooks (e.g. Clerk signing secret) in `.env` and never commit them.

## Best practices

- Use **HTTPS** for any publicly reachable service (Store, Integrations in production).
- Rotate **API keys** and **ADMIN_SECRET** if you suspect exposure; regenerate OAuth tokens via disconnect/reconnect if needed.
- Prefer **least privilege:** run the agent with a user that cannot access unrelated files; restrict `AGENT_WORKSPACE_ROOT` and `ALLOWED_BINARIES` to what the agent needs.
