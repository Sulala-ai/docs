## Overview

This page points you to where release and change information lives. For detailed version history, check the repositories and release tags.

## Where to find changes

- **Agent** — See the [Agent repository](https://github.com/schedra/sulala) (or your fork). Releases and tags (e.g. `v0.1.13`) often list fixes and new features. The agent’s `package.json` version is the runtime version when installed from npm.
- **Integrations** — Same repo or a dedicated integrations repo; check `integrations/package.json` and release notes.
- **Store** — Same repo or store-specific repo; check `store/package.json` and any `CHANGELOG.md` or GitHub Releases.

## What we track here (summary)

- **Agent:** Gateway, file watcher, task scheduler, AI orchestration, skills (registry install, tools.yaml), agent runner (sessions, streaming, tools), dashboard, desktop app, CLI.
- **Integrations:** New OAuth providers, store backend (file vs database), API auth (INTEGRATIONS_API_SECRET), Bluesky and other atproto/oauth flows.
- **Store:** Registry and skill API, submissions, admin, creator accounts (Clerk), paid skills (Stripe Connect), sync from repo, storefront and discovery.

When a new major or minor version adds behaviour that affects the docs (e.g. new env var, new API endpoint), we update the relevant doc page and can note it here in a short “Recent” subsection if needed.
