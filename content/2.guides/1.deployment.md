## Agent

### Run as a daemon (recommended for “always on”)

After installing the CLI (`npm i -g @sulala/agent`):

```bash
sulala onboard
sulala onboard --install-daemon
sulala start
```

This installs a background service (launchd on macOS, systemd on Linux) so the agent runs at login. Logs: `~/.sulala/logs/`. Use `sulala stop` / `sulala start` to control it; `sulala onboard --uninstall-daemon` to remove.

### Docker

From the agent directory:

```bash
docker compose up --build
```

Opens http://localhost:2026. Mount `./config` for watched folders; data lives in a named volume. Set env via `.env` or `environment` in `docker-compose.yml`.

### Production checklist

- Set **`GATEWAY_API_KEY`** if the gateway is reachable (e.g. behind a reverse proxy).
- Use a dedicated **`.env`** with all API keys and `HOST`/`PORT` as needed.
- Keep **`HOST=127.0.0.1`** if only local access is desired; bind to a different address only if you explicitly want remote access and have secured the gateway.

## Integrations

- **Run:** Same Node app; typically run on a host or container that is reachable over HTTPS for OAuth callbacks.
- **`INTEGRATIONS_CALLBACK_BASE_URL`:** Must be the **exact** public URL where the service lives (e.g. `https://integrations.yourdomain.com`). Each OAuth provider must have this callback URL registered in its app settings.
- **`INTEGRATIONS_DATA_PATH`:** Where tokens are stored. Use an absolute path and ensure the process has read/write access; back it up if needed.
- **Secrets:** Put all provider client IDs and secrets in `.env`; never in code or repo.

## Store

- **Vercel:** Connect the repo and deploy. Set **`NEXT_PUBLIC_STORE_URL`** to your production URL (e.g. `https://your-store.vercel.app`) so install instructions and registry `url` fields are correct. On Vercel the filesystem is read-only—admin writes to `data/` won’t persist; use GitHub sync or a self-hosted instance for persistent registry/submissions.
- **Self-hosted:** `pnpm build` then `pnpm start`. Set **`NEXT_PUBLIC_STORE_URL`** to your public base URL. Ensure the app can write to `data/` if you use the web form or admin to add/approve skills.
- **Database (optional):** For creator accounts and paid skills, set up Supabase (or your DB), run migrations from `store/supabase/`, and configure Clerk webhooks and Stripe as in the Store README.
- **Admin:** Set **`ADMIN_SECRET`** in production so `/admin` is protected.
