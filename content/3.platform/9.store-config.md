## Overview

Copy `store/.env.example` to `store/.env` and set the variables you need. Keep secrets out of git.

## Base URL

| Variable | Description |
|----------|-------------|
| **NEXT_PUBLIC_STORE_URL** | Base URL of this store (used in registry API and install instructions). **Required for production.** Defaults to http://localhost:3000 in dev. Set to your public URL (e.g. `https://hub.sulala.ai`). |

## Admin

| Variable | Description |
|----------|-------------|
| **ADMIN_SECRET** | Secret to protect `/admin` (login, approve/reject submissions, remove skills). Required if you use the admin panel. Omit or leave empty to disable admin. |

## Creator accounts (Clerk, optional)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key. Get at https://dashboard.clerk.com. Enable Google under User & Authentication тЖТ Social connections. |
| `CLERK_SECRET_KEY` | Clerk secret key. |
| `CLERK_WEBHOOK_SIGNING_SECRET` | Optional. For syncing signed-in users to DB. Clerk Dashboard тЖТ Webhooks тЖТ Endpoint тЖТ Signing secret. Webhook URL: `https://your-domain.com/api/webhooks/clerk`. |

Without Clerk keys, the store runs in keyless mode (no sign-in); you can still browse and submit skills.

## Database (Supabase, optional)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase project URL (e.g. `https://YOUR_PROJECT_REF.supabase.co`). |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase service role key. |

Required for storing users synced from Clerk webhooks and for **paid skills** (purchases, licenses, Stripe Connect). Run `store/supabase/migrations/` (including `20250227100000_paid_skills.sql`). From `store/`: `pnpm db:push` or Supabase CLI.

## Paid skills (Stripe, optional)

| Variable | Description |
|----------|-------------|
| `STRIPE_SECRET_KEY` | Stripe secret key (`sk_live_...` or `sk_test_...`). Required for Stripe and Connect server-side API calls. Get at https://dashboard.stripe.com/apikeys. |
| `NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY` | Optional; only needed if you add client-side Stripe (e.g. Elements). Current Checkout flow may not use it. |
| `STRIPE_WEBHOOK_SECRET` | Optional. To record purchases after checkout. Dashboard тЖТ Developers тЖТ Webhooks тЖТ endpoint URL `https://your-domain.com/api/webhooks/stripe`, event `checkout.session.completed`. Copy signing secret. |
| `STRIPE_CONNECT_CLIENT_ID` | Optional. Used for Standard/OAuth Connect flows. Express onboarding uses `STRIPE_SECRET_KEY` only. |

Enable supported countries at Stripe Dashboard тЖТ Settings тЖТ Connect тЖТ Express (Country and Capabilities).

## Submit and discovery

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_SUBMIT_GITHUB_ISSUE_URL` | Optional. GitHub issue URL for тАЬSubmit via GitHubтАЭ on `/submit` (e.g. `https://github.com/owner/repo/issues/new?template=skill.md`). |
| `SUBMIT_API_KEY` | Optional. If set, only requests with this key (or per-creator publish keys from My skills) can submit via API. If neither this nor creator keys are set, store may accept unauthenticated submissions. |
