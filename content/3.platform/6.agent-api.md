## Base URL and auth

- **Base URL:** Gateway origin (e.g. `http://127.0.0.1:2026`).
- **Auth:** When `GATEWAY_API_KEY` is set, send `X-Api-Key: <key>` or `?api_key=<key>` on requests.

## Sessions

| Method | Path | Description |
|--------|------|-------------|
| **GET** | `/api/agent/sessions` | List sessions. Query: `limit` (default 50). Returns `{ sessions: [{ id, session_key, meta, created_at, updated_at }] }`. |
| **POST** | `/api/agent/sessions` | Create or get session. Body: `{ "session_key": "my-chat", "meta": {} }`. Same `session_key` returns existing session. |
| **GET** | `/api/agent/sessions/:id` | Get session and message history. Returns `{ id, session_key, meta, created_at, updated_at, messages: [{ role, content, tool_calls?, tool_call_id?, name?, created_at }] }`. |

## Messages (run agent turn)

| Method | Path | Description |
|--------|------|-------------|
| **POST** | `/api/agent/sessions/:id/messages` | Send a user message and run the agent turn (model may call tools; loop runs until no tool calls or max turns). Body: `{ "message", "system_prompt?", "provider?", "model?", "max_tokens?", "timeout_ms?", "use_pi?" }`. Returns `{ sessionId, messages, finalContent, turnCount }`. |
| **POST** | `/api/agent/sessions/:id/messages/stream` | Same as above but streams the assistant reply. Events: `start` (data: `{ runId }`), `assistant` (data: `{ delta }`), `tool_call` (data: `{ name, result? }`), `done` (data: `{ finalContent, turnCount }`), `error` (data: `{ message }`). Also broadcast on WebSocket as `{ type: 'agent_stream', runId, event, data }`. |
| **POST** | `/api/agent/run` | Start an agent run in the background. Body: `{ "session_id" or "session_key", "message", "system_prompt?", "provider?", "model?", "max_tokens?", "timeout_ms?" }`. Returns **202** `{ runId, sessionId }`. Stream events only via WebSocket. |

## Behaviour

- **Per-session queue:** Only one turn runs per session at a time; further requests for the same session wait.
- **Timeout:** Set `AGENT_TIMEOUT_MS` in env or `timeout_ms` in the request body. On exceed or client disconnect, the run is aborted. Non-stream returns **499**; stream sends an `error` event.
- **Models:** Use `provider` (e.g. `openai`, `openrouter`, `claude`, `gemini`, `ollama`) and optional `model`. Defaults come from env (e.g. `AI_OPENAI_DEFAULT_MODEL`).
- **Context:** `AGENT_MAX_HISTORY_MESSAGES` and `AGENT_MAX_CONTEXT_TOKENS` limit how much history is sent to the model; older messages are dropped.
- **Tools:** `AGENT_TOOL_ALLOWLIST` (comma-separated names) restricts which tools are available. `AGENT_TOOL_PROFILE`: `full` | `messaging` | `coding` | `minimal` (e.g. messaging = run_task only; coding adds read_file, write_file, run_command).
- **Tool retry:** `AGENT_TOOL_RETRY_COUNT` (default 2); failed tool calls are retried with backoff before returning error to the model.

## Models list

| Method | Path | Description |
|--------|------|-------------|
| **GET** | `/api/agent/models?provider=openrouter` | List models for OpenRouter (proxies OpenRouter API). Returns `{ models: [{ id, name }] }`. Other providers may return `{ models: [] }`. |

## Skills API (agent)

| Method | Path | Description |
|--------|------|-------------|
| **GET** | `/api/agent/skills` | List installed skills. |
| **GET** | `/api/agent/skills/registry` | Fetch registry from `SKILLS_REGISTRY_URL`. |
| **POST** | `/api/agent/skills/install` | Install a skill. Body: `{ "slug", "target?" }`. |
| **POST** | `/api/agent/skills/uninstall` | Uninstall a skill. Body: `{ "slug", "target?" }`. |
| **POST** | `/api/agent/skills/publish` | Submit a skill to the store. Body: `{ "slug", "priceIntent?", "intendedPriceCents?" }`. |
| **POST** | `/api/agent/skills/update` | Refresh installed skills from the registry. |

## Built-in tools (used by the model)

- **run_task** — Enqueue a background task. Params: `type`, `payload` (optional). Returns `{ taskId, type, status }`.
- **read_file** — Read a file under `AGENT_WORKSPACE_ROOT`. Params: `path` (relative). Only registered when `AGENT_WORKSPACE_ROOT` is set.
- **write_file** — Write a file under the workspace (when enabled).
- **run_command** — Run a binary with args. Params: `binary`, `args` (array). Restrict with `ALLOWED_BINARIES` in config.
- **run_agent** — Call another session (sub-agent), one level only.
