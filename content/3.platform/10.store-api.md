## Overview

The Store exposes endpoints for the **agent** (registry + skill content) and optional **submissions** and **admin**. The agent expects the registry and skill URLs to follow this contract.

## Agent-facing (registry and skills)

### GET /api/sulalahub/registry

Returns the list of skills available for install. The agent fetches this from `SKILLS_REGISTRY_URL` (e.g. `https://hub.sulala.ai/api/sulalahub/registry`).

**Response:** JSON with a `skills` array. Each entry should include:

| Field | Description |
|-------|-------------|
| `slug` | Unique identifier (e.g. `news`, `apple-notes`). Used in `sulala skill install <slug>`. |
| `name` | Display name. |
| `description` | Short description for the catalog and agent. |
| `version` | Optional. Semver string. |
| `url` | **Required.** Full URL to fetch the skill content (Markdown). Usually `NEXT_PUBLIC_STORE_URL/api/sulalahub/skills/<slug>` or with `?license=TOKEN` for paid skills. |

Example:

```json
{
  "skills": [
    {
      "slug": "news",
      "name": "News",
      "description": "Fetch and summarize news.",
      "version": "1.0.0",
      "url": "https://hub.sulala.ai/api/sulalahub/skills/news"
    }
  ]
}
```

### GET /api/sulalahub/skills/:slug

Returns the raw skill content (Markdown). Optional query:

| Query | Description |
|-------|-------------|
| `license` | For paid skills, the buyer’s license token. Required to access paid skill content. |

**Response:** Raw Markdown body (e.g. from `data/skills/<slug>/README.md`). 404 if slug not found or (for paid skills) license missing/invalid.

### GET /api/sulalahub/skills/:slug/tools

Optional. Returns the skill’s `tools.yaml` content if present. Used by the agent when installing from the hub. 404 if no tools.yaml.

## Submissions (optional)

### POST /api/submissions

Submit a skill for inclusion in the store. Used by the dashboard “Publish to store” and by external tools.

**Headers:** Optional `Authorization: Bearer <key>` or `X-Api-Key: <key>` if the store requires auth (`SUBMIT_API_KEY` or per-creator publish key).

**Body:** JSON with at least:

| Field | Description |
|-------|-------------|
| `slug` | Skill slug. |
| `name` | Display name. |
| `description` | Short description. |
| `content` | Full skill Markdown (README body). |
| `toolsYaml` | Optional. YAML string for custom tools. |

On success, the submission is stored (e.g. in `data/submissions.json` or DB) for admin review. On Vercel (read-only filesystem), submission may fail; store can point users to “Submit via GitHub” using `NEXT_PUBLIC_SUBMIT_GITHUB_ISSUE_URL`.

## Admin (optional)

Admin endpoints are protected by `ADMIN_SECRET`. Client must authenticate (e.g. cookie set after login at `/admin` with the secret).

- **Approve submission** — Adds skill to registry and writes content to `data/skills/<slug>/README.md` (and optional `tools.yaml`).
- **Reject submission** — Removes from queue.
- **Remove skill** — Removes from registry and optionally deletes skill files.

Exact paths and request shapes are implementation-specific; see the store app (e.g. `store/app/api/`) for details.

## Summary

| Consumer | Endpoint | Purpose |
|----------|----------|---------|
| Agent | `GET /api/sulalahub/registry` | List skills, get `url` for each. |
| Agent | `GET /api/sulalahub/skills/:slug` | Fetch skill Markdown (optional `?license=` for paid). |
| Agent | `GET /api/sulalahub/skills/:slug/tools` | Fetch tools.yaml. |
| Dashboard / tools | `POST /api/submissions` | Submit a skill. |
| Admin UI | Admin APIs | Approve, reject, remove. |
