## Overview

The **Sulala CLI** is the command-line interface for the Agent. Use it when the agent is installed globally (`npm i -g @sulala/agent`) as `sulala`, or from the repo as `npm run cli -- <args>`.

**Environment:** The CLI talks to the gateway. Set `GATEWAY_URL` (default `http://127.0.0.1:2026`) and optionally `GATEWAY_API_KEY` if the gateway requires auth. For skill commands, `SULALA_SKILLS_DIR` and `AGENT_CONTEXT_PATH` can affect paths; `SKILLS_REGISTRY_URL` is used for `skill install` and `skill list` (registry fetch).

## Commands

### status

```bash
sulala status
```

Calls the gateway `/health` and prints the JSON response. Use to check that the agent is running.

### doctor

```bash
sulala doctor
```

Runs basic checks: env (PORT, AI keys or Ollama), DB existence, gateway reachable, skills fetch. Prints `ok` and `issues` arrays; exits with code 1 if there are issues.

### tasks

```bash
sulala tasks [--limit=50]
```

Fetches recent tasks from the gateway (`/api/tasks`) and prints JSON. Default limit 50.

### logs

```bash
sulala logs [--limit=100]
```

Fetches recent logs from the gateway (`/api/logs`) and prints JSON. Default limit 100.

### enqueue

```bash
sulala enqueue --type=NAME [--payload='{"key":"value"}']
```

Enqueues a task. `--type` is required; `--payload` is optional JSON. Useful for triggering task types from scripts.

### skill list

```bash
sulala skill list
```

Fetches the skills registry (from `SKILLS_REGISTRY_URL`) and prints the list of available skills (slug, name, description, etc.) as JSON.

### skill check

```bash
sulala skill check
```

Checks installed skills and required binaries. Prints counts (skills, eligible, blocked, unknown) and `requiredBins`. Exits with 1 if any skill is blocked (missing binary).

### skill install

```bash
sulala skill install <slug> [--global] [--registry=URL]
sulala skill install --from-url=URL [--global]
```

Installs a skill by **slug** from the registry (default registry from `SKILLS_REGISTRY_URL`), or from a **URL** (e.g. paid skill install URL with `?license=...`). Installs to workspace skills directory (~/.sulala/workspace/skills). Use `--registry=URL` to override the registry for this install. `--global` targets the managed skills dir.

### skill uninstall

```bash
sulala skill uninstall <slug> [--global]
```

Removes the skill from workspace (default) or managed (`--global`) skills directory.

### skill update

```bash
sulala skill update [--all]
```

Refreshes installed skills from the registry (re-fetches README and tools.yaml). Use `--all` to update all installed skills. Prints result (updated/failed). Exits with 1 if any update failed.

### init

```bash
sulala init [dir]
```

Creates `config/` and `context/` in the given directory (default: current directory). Copies `.env.example` to `.env` if `.env` does not exist. Run from the agent repo root to get the agent’s `.env.example`.

### onboard

```bash
sulala onboard [--install-daemon] [--uninstall-daemon] [--reset]
```

- **onboard** — Ensures `~/.sulala` exists and opens the onboarding page in the browser (http://127.0.0.1:2026/onboard or your PORT). Use this to add AI API keys and complete first-run setup.
- **onboard --install-daemon** — Same as above, then installs the background daemon (launchd on macOS, systemd on Linux) so the agent runs at login. Opens the onboarding page after a short delay.
- **onboard --uninstall-daemon** — Removes the daemon (no longer starts at login).
- **onboard --reset** — Resets onboarding state so the next dashboard load shows the onboarding flow again.

### start

```bash
sulala start
```

Starts the agent daemon (if it was installed with `onboard --install-daemon`).

### stop

```bash
sulala stop
```

Stops the agent daemon.

### version

```bash
sulala version [--check]
sulala --version
sulala -V
```

Prints the CLI version. With `--check`, compares to the latest version on npm and suggests an upgrade if different.

## Environment variables (CLI)

| Variable | Description |
|----------|-------------|
| `GATEWAY_URL` | Gateway base URL (default `http://127.0.0.1:2026`). CLI commands that call the gateway use this. |
| `GATEWAY_API_KEY` | If set, sent as `X-Api-Key` on gateway requests. |
| `SKILLS_REGISTRY_URL` | Used by `skill list` and `skill install` to fetch the registry and skill content. |
| `SULALA_SKILLS_DIR` | Override for skills directory (e.g. workspace skills path). |
| `AGENT_CONTEXT_PATH` | Override for agent context path (used by some skill resolution). |

## Examples

```bash
# Check agent is up
sulala status

# Run health check
sulala doctor

# List available skills from registry
sulala skill list

# Install a skill
sulala skill install news

# Install from URL (e.g. paid skill with license)
sulala skill install --from-url="https://hub.sulala.ai/api/sulalahub/skills/paid-skill?license=TOKEN"

# Update installed skills
sulala skill update

# First-time setup and install daemon
sulala onboard --install-daemon

# Show version and check for updates
sulala version --check
```
