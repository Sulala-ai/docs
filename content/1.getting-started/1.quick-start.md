## Goal

Run the **Sulala Agent** on your machine, open the dashboard, optionally connect one integration (e.g. GitHub), and install one skill from the **SulalaHub Store**.

## Step 1: Install and start the Agent

**Option A — One-line install (macOS / Linux):**

```bash
curl -fsSL https://sulala.ai/install.sh | bash
```

**Option B — npm:**

```bash
npm i -g @sulala/agent
sulala onboard
sulala start
```

**Option C — From repo:**

```bash
cd agent
npm install
cp .env.example .env
npm start
```

When the agent is running, open **http://127.0.0.1:2026**. You should see the dashboard. On first run, onboarding may prompt you to add AI API keys (Ollama works with no key).

## Step 2: (Optional) Run Integrations and connect an account

If you want the agent to use Gmail, GitHub, Slack, etc., run the **Integrations** service and connect at least one provider.

1. **Start Integrations** (in another terminal):

   ```bash
   cd integrations
   npm install
   npm run dev
   ```

   Default URL: http://127.0.0.1:1717

2. **Configure the agent to use Integrations:**  
   In the agent dashboard go to **Settings → Portal** (or Integrations). Set:
   - **Portal / Integrations URL** to `http://127.0.0.1:1717` (or your deployed integrations URL)
   - **API key** if your integrations service requires one

3. **Connect a provider:**  
   In the dashboard open **Integrations** (or the connections area). Click **Connect** for a provider (e.g. GitHub). Complete OAuth in the browser. The dashboard will store a `connectionId`; the agent will use it when running tools that need that provider.

## Step 3: Point the Agent at the Store and install a skill

1. **Set the skills registry:**  
   In the agent, set `SKILLS_REGISTRY_URL` to your store’s registry endpoint, for example:
   - Public hub: `https://hub.sulala.ai/api/sulalahub/registry`
   - Local store: `http://localhost:3000/api/sulalahub/registry` (if you run the store locally)

   You can set this in `.env` or in the dashboard (Settings) if the UI exposes it.

2. **Install a skill:**  
   - **Dashboard:** Open **Skills** → **Add skill from registry** → pick a skill (e.g. **news**) → Install.  
   - **CLI:** `sulala skill install news` (or another slug from the registry).

3. **Use the agent:**  
   In the dashboard chat (or via API), ask something that matches the skill’s description. The agent will use the skill’s instructions and tools (e.g. `run_task`, `run_command`) to answer.

## Step 4: (Optional) Run the Store locally

To run the **SulalaHub Store** yourself (e.g. to browse skills or submit your own):

```bash
cd store
pnpm install
cp .env.example .env
# Set NEXT_PUBLIC_STORE_URL=http://localhost:3000 in .env
pnpm dev
```

Open http://localhost:3000. To install skills from this local store, set the agent’s `SKILLS_REGISTRY_URL` to `http://localhost:3000/api/sulalahub/registry`.
