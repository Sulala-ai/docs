## Overview

The **Sulala Agent** is a local-first platform that runs on `127.0.0.1` and combines **file monitoring**, **task scheduling**, **AI orchestration**, and a **plugin/skill system**. It stays on your machine and does not expose services to the internet by default.

## Architecture

| Layer | Role |
|-------|------|
| **Gateway** | REST + WebSocket API on `localhost:2026`; auth and request handling |
| **File watcher** | Real-time folder watch (add/change/delete) → event triggers |
| **Task scheduler** | Cron-like scheduling + queue with retries and failure handling |
| **AI orchestration** | Single interface to multiple providers (OpenAI, OpenRouter, Claude, Gemini, Ollama); routing and rate limits |
| **Plugins / skills** | Scripts and integrations that hook into events, tasks, and AI |
| **Persistence** | SQLite for tasks, file state, logs, and AI results |

All traffic stays on `127.0.0.1` unless you explicitly configure otherwise. Credentials live in `.env`; never in the repo.

## Quick start

**One-line install:**

- **macOS / Linux:** `curl -fsSL https://sulala.ai/install.sh | bash`
- **Windows (PowerShell):** `irm https://sulala.ai/install.ps1 | iex`

This installs the CLI, runs onboarding, and can install the background daemon. Dashboard: **http://127.0.0.1:2026**

**From npm:**

```bash
npm i -g @sulala/agent
sulala onboard
sulala onboard --install-daemon
sulala start
```

**From source (repo):**

```bash
cd agent && npm install && cp .env.example .env
npm start
```

Development: `npm run dev` (restarts on file changes). Dashboard dev: `npm run dashboard` (UI on its own port; set `VITE_GATEWAY_URL=http://127.0.0.1:2026` in `dashboard/.env` if needed).

## CLI

```bash
sulala status
sulala tasks --limit=20
sulala logs --limit=50
sulala skill list
sulala skill install <slug> [--global]
sulala skill update
sulala skill uninstall <slug> [--global]
sulala init [dir]
```

- **Onboard:** `sulala onboard` opens the browser to add API keys; `sulala onboard --install-daemon` installs a background service (launchd on macOS, systemd on Linux).
- **Config:** `~/.sulala` for global config and data; optional project `.sulala/config.json` when running from that directory.

For all commands, options, and env vars, see [CLI reference](/platform/cli-reference).

## Skills

Skills are **documentation-driven**: Markdown (and optional tool specs) that the agent loads as context. No per-skill code in the agent—you add skills by adding or editing files and pointing the agent at a **skills registry**.

- **Registry:** Set `SKILLS_REGISTRY_URL` (e.g. `https://hub.sulala.ai/api/sulalahub/registry`) to list and install skills from a hub.
- **Install:** Dashboard → Skills → Add skill from registry, or CLI: `sulala skill install <slug>`.
- **Paths:** User (`~/.sulala/workspace/skills/`) > installed (hub) > workspace > managed (`~/.sulala/skills`) > extra dirs. No built-in system skills; install from the hub.

## Agent API (sessions & chat)

Multi-turn sessions with optional tool use:

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/agent/sessions` | List sessions (`?limit`) |
| POST | `/api/agent/sessions` | Create or get session (`session_key`, `meta`) |
| GET | `/api/agent/sessions/:id` | Get session and message history |
| POST | `/api/agent/sessions/:id/messages` | Send message and run agent turn (optional streaming) |

Runs are serialized per session; timeout and tool profiles are configurable via env or request body. See the agent’s [Agent API](https://github.com/schedra/sulala/blob/main/agent/docs/agent-api.md) and [tools](https://github.com/schedra/sulala/blob/main/agent/docs/tools.md) docs for details.

## AI models

Supported providers: **OpenAI**, **OpenRouter**, **Claude**, **Gemini**, **Ollama** (local). Default is Ollama (no API key; installer runs if missing). Override with `AI_DEFAULT_PROVIDER` and set the corresponding API keys in onboarding or `.env`.

## Config

- **Watched folders:** `.env` (`WATCH_FOLDERS`) or `config/watched.json` (array `folders`); merged.
- **Gateway auth:** `GATEWAY_API_KEY`.
- **Webhooks:** `WEBHOOK_URL` or `WEBHOOK_URLS`, optional `WEBHOOK_SECRET`.
- **Portal / integrations:** Set `PORTAL_GATEWAY_URL` and `PORTAL_API_KEY` so the agent can list OAuth connections and get tokens for tools (Gmail, Slack, GitHub, etc.).

## Desktop app

Installable app (Mac/Windows) that runs the gateway and opens the dashboard in a window. From `agent/`: `npm run build && npm run dashboard:build && npm run desktop`. Installers: `npm run desktop:pack:mac` (DMG), `npm run desktop:pack:win` (NSIS/portable).

## Requirements

- **Node.js** 18+
- **TypeScript** + **tsx** (dev)
- **Python 3.10+** (optional, for automation scripts)
- SQLite via **sql.js** (WASM; no native addon)
