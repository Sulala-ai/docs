## Overview

The Agent dashboard runs at **http://127.0.0.1:2026** (or your configured port). It‚Äôs the main place to manage skills, connect OAuth providers, chat with the agent, and change settings.

## Main areas

### Skills

- **List installed skills** ‚Äî See which skills are loaded (from workspace and hub).
- **Add skill from registry** ‚Äî If `SKILLS_REGISTRY_URL` is set, browse the registry and install a skill by slug. Installs to your workspace skills directory.
- **Update all** ‚Äî Refresh installed skills from the registry (e.g. pull latest README and tools.yaml).
- **Enable/disable** ‚Äî Turn a skill on or off without uninstalling (stored in config).
- **Publish to store** ‚Äî For skills you created (e.g. under ‚Äúmy‚Äù workspace), submit them to the store for others to install. Requires store URL (derived from `SKILLS_REGISTRY_URL`) and optionally a publish API key.
- **Refresh** ‚Äî After adding or editing a skill on disk, use Refresh (or restart the gateway) so the agent picks it up. With `SKILLS_WATCH=1` (default), changes may be detected automatically.

### Integrations / Connections

- **Connect providers** ‚Äî Start OAuth for a provider (GitHub, Gmail, Slack, etc.). You‚Äôre redirected to the provider to sign in; after approval, the service stores a connection and the dashboard shows a `connectionId`. The agent uses that connection when running tools that need that provider.
- **Disconnect** ‚Äî Remove a connection (deletes stored tokens for that connection).
- **List connections** ‚Äî See which providers are connected; tokens are never shown in the UI.

Requires the Integrations service (or Portal) to be running and the agent to be configured with its URL and API key (see [Agent configuration](/platform/agent-config)).

### Chat

- **Sessions** ‚Äî Each chat is a session. You can start a new session or continue an existing one.
- **Send messages** ‚Äî The agent uses loaded skills and tools to answer. If you have execution preview enabled (`AGENT_EXECUTION_PREVIEW=1`), high-risk actions (e.g. `write_file`, `run_command`, `post_to_bluesky`) may appear in **Pending** for approval before they run.
- **Create a skill via chat** ‚Äî Ask the agent to create a skill (e.g. ‚ÄúCreate a hello world skill‚Äù). It will create the folder with README.md and tools.yaml in your workspace skills directory.

### Settings

- **Portal / Integrations** ‚Äî Set the Integrations (or Portal) gateway URL and API key so the agent can list connections and get tokens. Optionally Store publish API key for ‚ÄúPublish to store.‚Äù
- **AI / Onboarding** ‚Äî Add or change AI API keys (OpenAI, Claude, Gemini, OpenRouter). Default is Ollama (no key).
- **Other** ‚Äî Depending on the build, you may see options for execution preview, workspace root, or registry URL.

### Pending (when execution preview is on)

If `AGENT_EXECUTION_PREVIEW=1`, high-risk tool calls appear here. Approve or reject before the agent continues. Useful for safety when the agent can run commands or write files.

## Tips

- **Skills not showing?** Ensure `SKILLS_REGISTRY_URL` is set and the agent can reach it. Refresh skills or restart the gateway.
- **Integration ‚Äúinvalid token‚Äù?** Confirm Portal/Integrations URL and API key in Settings, and that you completed OAuth (connect) for that provider.
- **Dashboard not loading?** Ensure the agent is running (`npm start` or `sulala start`) and you‚Äôre opening the correct port (default 2026).
