## Agent

- **Gateway** — The HTTP/WebSocket server (default `127.0.0.1:2026`) that serves the dashboard, REST API, and agent sessions. Single entry point on the user’s machine.
- **Session** — A conversation container. Identified by `id` or `session_key`. One agent “turn” runs at a time per session; messages and tool calls are stored with the session.
- **Turn** — One user message plus the model’s reply (and any tool calls and retries) until the model stops or hits limits.
- **Skill** — A documentation-only Markdown resource (with optional frontmatter) that describes when and how to use tools. Loaded from user dir, workspace, managed dir, bundled context, or the store. No per-skill code in the agent.
- **Tool** — A callable capability the model can use (e.g. `run_task`, `read_file`, `run_command`). Tools are generic; skills describe which commands or actions to run.
- **Registry** — A list of skills (slug, name, description, url) fetched from `SKILLS_REGISTRY_URL`. Used to list and install skills from a hub/store.
- **Config path** — The JSON file used for skill enable/disable and per-skill options. Resolved via `SULALA_CONFIG_PATH` or default `~/.sulala/config.json` (or `.sulala/config.json` in cwd if it exists).

## Integrations

- **Provider** — An OAuth-backed service (e.g. GitHub, Gmail, Slack). Each has an entry in the Integrations API (`/providers`, `/integrations`).
- **Connection** — A user’s linked account for a provider. Stored in the Integrations service; identified by **connectionId**. The agent (or any app) uses `connectionId` to request an access token via `POST /connections/:id/use`.
- **connectionId** — Unique id for a connection. Apps store this (e.g. in skill config or agent settings); they never store the actual OAuth tokens. Tokens are fetched at runtime from the Integrations service.

## Store

- **Store / SulalaHub Store** — The web app that hosts the skill registry and catalog. Users set `SKILLS_REGISTRY_URL` to the store’s registry endpoint so the agent can list and install skills.
- **Registry (store)** — The JSON list of skills returned by `GET /api/sulalahub/registry`. Each entry includes `slug`, `name`, `description`, `version`, and `url` (link to fetch the skill Markdown).
- **Slug** — Short identifier for a skill (e.g. `news`, `apple-notes`). Used in URLs, install commands (`sulala skill install <slug>`), and the registry.
- **Submission** — A skill submitted for inclusion in the store (e.g. via web form or publish API). Stored in a queue; admin approves or rejects.
- **License (paid skills)** — Token issued after purchase of a paid skill. Required to fetch paid skill content (e.g. `GET /api/sulalahub/skills/:slug?license=TOKEN`).

## General

- **Portal** — In the agent, “Portal” often refers to the Integrations (or combined OAuth) service: the agent is configured with `PORTAL_GATEWAY_URL` and `PORTAL_API_KEY` to list connections and get tokens.
- **Onboarding** — First-run flow in the agent dashboard (e.g. add AI API keys, optional daemon install). Triggered by `sulala onboard` or opening the dashboard for the first time.
