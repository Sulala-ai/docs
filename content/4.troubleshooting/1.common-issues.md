## Common Issues

### Gateway not starting (Agent)

**Problem**: The agent gateway doesn’t start or you can’t open the dashboard at http://127.0.0.1:2026.

**Solutions**:

1. **Port in use** — Another process may be using port 2026. Stop it or set a different port in `.env`.
2. **Dependencies** — From `agent/` run `npm install` and try again.
3. **Config** — Copy `.env.example` to `.env` and adjust. Ensure `WATCH_FOLDERS` or `config/watched.json` exists if you use the file watcher.

### Skills not listing or installing (Agent)

**Problem**: Dashboard or CLI doesn’t show registry skills, or install fails.

**Solutions**:

1. **Registry URL** — Set `SKILLS_REGISTRY_URL` to your store’s registry (e.g. `https://your-store.vercel.app/api/sulalahub/registry`).
2. **Network** — Ensure the agent can reach the registry URL (no firewall blocking outbound HTTPS).
3. **Config path** — Agent uses `~/.sulala/config.json` by default, or `.sulala/config.json` in the current working directory if it exists. Check that the right config file is being used.

### Integrations OAuth callback fails

**Problem**: After signing in with a provider (e.g. GitHub), the redirect shows an error or doesn’t return to your app.

**Solutions**:

1. **Callback URL** — Set `INTEGRATIONS_CALLBACK_BASE_URL` to the exact URL where the integrations service is reachable (e.g. `http://localhost:1717` for local dev). The provider must have this callback URL configured in its OAuth app settings.
2. **Provider credentials** — Ensure the correct client ID and secret are in `.env` (e.g. `GITHUB_CLIENT_ID`, `GITHUB_CLIENT_SECRET`).

### Store registry or skill content not loading

**Problem**: The storefront or agent can’t load the registry or a skill’s Markdown.

**Solutions**:

1. **Base URL** — Set `NEXT_PUBLIC_STORE_URL` to the store’s public URL (e.g. `https://your-store.vercel.app`) so registry `url` fields and install instructions are correct.
2. **Data** — Ensure `data/registry.json` and `data/skills/<slug>/README.md` (or legacy `data/skills/<slug>.md`) exist. Run `pnpm sync-registry` from the store if you sync from the main repo.

### Agent can’t use integrations (no token)

**Problem**: Agent tools that need OAuth (Gmail, Slack, etc.) fail with missing or invalid token.

**Solutions**:

1. **Portal URL and API key** — In the agent dashboard (Settings → Portal), set the integrations service URL and API key so the agent can call `POST /connections/:id/use`.
2. **Connection in dashboard** — Connect the provider in the dashboard Integrations (or Portal) area first; then ensure the skill or tool config uses the correct `connectionId`.
