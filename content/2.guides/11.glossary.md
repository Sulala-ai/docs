## A–Z

**Agent** — The Sulala Agent: local process (gateway, file watcher, scheduler, AI orchestration, skills). Runs on your machine at `127.0.0.1:2026` by default.

**Auth (tools)** — In `tools.yaml`, how a tool gets credentials: `none`, `stripe_secret_key`, `discord_bot_token`, or `portal` (OAuth via Integrations/Portal).

**Connection** — A user’s linked OAuth account for a provider (e.g. GitHub). Stored in the Integrations service; identified by **connectionId**.

**connectionId** — Unique ID for a connection. The agent (or app) stores this and calls the Integrations API to get an access token when needed. Tokens are not stored in the agent.

**Gateway** — The HTTP/WebSocket server of the Agent (default port 2026). Serves the dashboard, REST API, and agent sessions.

**Integrations** — The OAuth connections service. Users connect providers (Gmail, GitHub, etc.) once; tokens are stored there. The agent uses **connectionId** to fetch tokens at runtime.

**Portal** — In the agent, “Portal” usually means the hosted Integrations gateway (e.g. portal.sulala.ai). The agent is configured with `PORTAL_GATEWAY_URL` and `PORTAL_API_KEY` to list connections and get tokens.

**Provider** — An OAuth-backed service (e.g. GitHub, Gmail, Slack). Each has a provider entry in the Integrations API and optional agent tools.

**Registry** — List of skills (slug, name, description, url) returned by the Store’s `/api/sulalahub/registry`. The agent fetches this from `SKILLS_REGISTRY_URL` to list and install skills.

**Session** — A conversation container in the agent. Identified by `id` or `session_key`. One agent turn runs at a time per session.

**Skill** — A folder with README.md (or SKILL.md) and optional tools.yaml. Teaches the agent when and how to use tools. No per-skill code; docs + optional YAML tools.

**Slug** — Short identifier for a skill (e.g. `news`, `helloworld`). Used in URLs, `sulala skill install <slug>`, and the registry.

**Store** — The SulalaHub Store: web app that hosts the skill registry and catalog. Users set `SKILLS_REGISTRY_URL` to the store’s registry endpoint.

**Tool** — A callable capability the model can use (e.g. `run_task`, `read_file`, `run_command`, or a custom tool defined in a skill’s tools.yaml).

**Turn** — One user message plus the model’s reply (and any tool calls) until the model stops or hits limits.

**tools.yaml** — Optional file in a skill folder. Defines custom tools (e.g. HTTP API calls) via a `tools:` array. No TypeScript required.

For more context, see [Concepts](/guides/concepts).
